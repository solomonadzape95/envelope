Envelope

Install Quick Start Features Commands

Share .env files without sharing secrets

Envelope locks your environment with symmetric encryption and distributes per-user lockboxes with asymmetric keys. No secret ever needs to live on a server you don't control.

Install Now Get Started

Installation

Install Envelope CLI globally via npm:

npm i -g envelope-cli

Or install from source:

npm run prepare && npm i -g .

Quick Start

  1. Generate your keys

    Creates a keypair and stores your username locally under ~/.envelope

    envelope gen
  2. Lock your .env

    Run in your project directory containing .env to encrypt it

    envelope lock
  3. Open (decrypt) your .env

    Decrypt and create a local .env file

    envelope open

Why Envelope? What's Different

Local-first trust model

Keys are generated and stored on your machine under ~/.envelope. Your raw secrets never need to be uploaded to any third-party service.

Double protection

Your .env is symmetrically encrypted, then the symmetric key is individually encrypted (lockboxes) with each teammate's public key.

Simple CLI, no servers

No dashboards or SaaS—just files in your repo and a small local database. Works offline and fits existing Git workflows.

Granular sharing

Share keys per project to specific users. Add or revoke recipients by updating lockboxes without re-sharing entire secrets.

Commands

gen

Generate local keypair and username.

envelope gen # options # --share <NAME> --project <REPO>

lock

Encrypt .env and create lockboxes for teammates.

envelope lock

open

Decrypt encrypted env into a local .env.

envelope open

share

Share your public key for a project with a team lead.

envelope share <project> <team_leader>

pull

Pull keys shared with you.

envelope pull <project>

update

Pull keys and re-lock your .env in one go.

envelope update <project>

Get help anytime:

envelope --help OR envelope <command> --help

How It Works

Key Storage

Public/private keys are stored locally in ~/.envelope. Team public keys are aggregated in .envelope/envelope_keys.txt.

Symmetric Encryption

Your .env is symmetrically encrypted to .envelope/envelope_enc.txt for fast, secure bulk encryption.

Asymmetric Lockboxes

The symmetric key is encrypted per-user (a lockbox) using their public key and stored in .envelope/envelopes.txt.

Secure Sharing

Team members can decrypt their lockbox with their private key to access the symmetric key and decrypt the shared .env.
